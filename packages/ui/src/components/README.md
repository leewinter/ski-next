# Components

Components are grouped using atomic design names where the distinction is useful:

- `atoms` - small reusable primitives such as buttons, inputs, icons, and labels.
- `molecules` - simple composed controls made from atoms.
- `organisms` - larger product components such as schedulers, forms, and data panels.
- `templates` - reusable page or workflow layouts, if the library needs them later.

Storybook titles should mirror these groups, for example `Atoms/Button` or `Organisms/TransportScheduler`.
